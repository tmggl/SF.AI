# Phase 27.35 — Live UI Trial Observations

## الهدف

اختبار تجربة الواجهة الحية كما سيستخدمها سامي:

- `/ui/chat` يحتوي زر `مولّد تجريبي`.
- الطلب يرسل `generator_trial=true`.
- المولّد الحقيقي `sf_10m_phase27_33` يرد على prompts المناسبة.
- المسار الافتراضي يبقى `template`.
- الهوية والمجالات الحساسة لا تذهب للمولّد.

## النتيجة

```text
status       : PASSED_LIVE_UI_TRIAL_READY_FOR_USER_OBSERVATION
ui_passed    : true
cases        : 10/10
generator    : 7/7
controls     : 3/3
```

## ما يعنيه ذلك

يمكن لسامي الآن اختبار المولّد من الواجهة فعليًا:

1. افتح:

```text
http://127.0.0.1:8123/ui/chat
```

2. فعّل زر:

```text
مولّد تجريبي
```

3. جرّب:

- `كيفك اليوم`
- `شكرًا لمساعدتك`
- `وجهني بخطوة بسيطة`
- `رتب لي يومي بسرعة`
- `توترت شوي وش اسوي`
- `وش المقصود بالاحترام`
- `القراية تفيدني بشي`

يجب أن يظهر في التشخيص:

```text
مولّد SF-10M Phase 27.33
```

## حدود المرحلة

- هذا trial محلي محروس، وليس runtime عام.
- الافتراضي بدون الزر يبقى template.
- أي مخرجات مولدة تبقى review evidence، وليست corpus تدريب.
- `SF-50M` ما زال محجوبًا حتى تنجح ملاحظات الواجهة فعليًا.
